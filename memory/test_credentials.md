# ItaliaModa AI Agent — Test Credentials

## Initial Admin Onboarding

⚠️ **На текущий момент пользователей в системе нет** — `/app/frontend/auth/users.json` пустой.

При первом открытии приложения вы увидите экран **"Создайте учётную запись администратора"** с полями:
- Логин (минимум 3 символа, только буквы/цифры/`.`/`_`/`-`)
- Пароль (минимум 8 символов)
- Повтор пароля

После создания этого первого админа эндпоинт `/api/auth/setup` **навсегда блокируется** (403). Новых пользователей можно создавать только через **админ-панель внутри приложения** (раздел "Пользователи" в drawer'е настроек).

## Тестовые сценарии

### Сценарий 1: Первый запуск (текущее состояние)
1. Открыть URL приложения
2. Должен показаться экран setup
3. Заполнить форму → нажать "Создать администратора"
4. Получить JWT в HttpOnly cookie + редирект на основной экран приложения

### Сценарий 2: Создание обычного пользователя (после setup)
1. Войти как admin
2. Открыть drawer настроек (значок шестерёнки в шапке)
3. Раздел "Пользователи" → "Создать пользователя"
4. Указать логин/пароль/роль (user или admin)
5. Передать новому пользователю эти креденшалы лично

### Сценарий 3: Lockout-защита
- 5 неудачных попыток входа подряд → блокировка аккаунта на 15 минут
- Rate limit: 10 попыток входа с одного IP за 15 минут

## Безопасность хранилища

- Пароли хешируются bcrypt cost=12 в `/app/frontend/auth/users.json`
- JWT подписан HS256 секретом из `JWT_SECRET` env (генерируется `openssl rand -hex 32`)
- Сессии живут 30 дней (HttpOnly + SameSite=Lax cookie)
- Подмена конкурентных записей в JSON предотвращается через `proper-lockfile`

## Сброс к заводским настройкам

Если потеряли пароль админа или нужно начать заново:
```bash
echo '{"users":[]}' > /app/frontend/auth/users.json
sudo supervisorctl restart frontend
```
При следующем открытии снова покажется экран setup.
